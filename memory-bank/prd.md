# 产品需求文档（PRD）

## 企业内部多智能体对话机器人

**版本**: v1.0
**日期**: 2026-04-27
**状态**: 草案

---

## 1. 概述

### 1.1 产品背景

企业内部存在大量重复性咨询和查询需求（规章制度、技术文档、业务数据等），分散在不同系统中，员工获取信息成本高。

### 1.2 核心目标

一个 Web 对话机器人，后端根据用户意图，将请求路由到不同 Agent（简单对话、RAG 知识检索、SQL 数据查询），统一回答员工问题。

### 1.3 目标用户

企业内部全体员工，覆盖日常咨询、技术文档查询、业务数据查询等场景。

---

## 2. 核心功能列表

### 2.1 基础能力

| 功能 | 说明 |
|------|------|
| Web 对话界面 | 支持文本输入、对话历史展示、新建/清空会话 |
| 意图识别 | 分析用户输入，判断应路由到哪个 Agent |
| Agent 路由 | 根据意图将请求分发给对应 Agent 处理 |
| 流式响应 | 支持 SSE 流式输出，提升交互体验 |

### 2.2 三大 Agent

| Agent | 触发场景 | 能力说明 |
|-------|---------|---------|
| **简单对话 Agent** | 日常寒暄、通用问答、无需外部数据的对话 | 基于大模型直接生成回答 |
| **RAG Agent** | 查询企业知识库（制度、文档、手册等） | 向量检索相关知识片段，结合 LLM 生成带引用来源的答案 |
| **Service SQL Agent** | 查询业务数据（如销售额、用户量、订单数等） | 调用后端 Service，将自然语言转为 SQL 查询数据库，返回结构化数据 |

### 2.3 交互流程

```
用户输入 → 意图识别 → Agent 路由 → Agent 处理 → 返回结果
              ↓
    ┌─────────┼─────────┐
    ↓         ↓         ↓
 简单对话   RAG检索   SQL查询
  Agent     Agent     Agent
```

### 2.4 意图识别规则示例

| 用户输入示例 | 识别意图 | 路由 Agent |
|-------------|---------|-----------|
| "你好" / "今天天气怎么样" | 通用对话 | 简单对话 Agent |
| "请假流程是什么" / "如何申请报销" | 知识查询 | RAG Agent |
| "上个月销售额多少" / "北京区用户量" | 数据查询 | Service SQL Agent |
